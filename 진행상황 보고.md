# 업무 팔로업 에이전트 — 진행 보고

> 2026-05-24 기준

## 팀 요청 vs 현재 MVP 차이

팀 요청은 "업무 팔로업 AI 에이전트" — 범위가 넓음. 첫 단계로 **카톡 단일 채널 MVP** 부터 검증했음. 이메일·캘린더·다른 메신저는 이번 범위 밖이고, 카톡에서 동작·신뢰도 검증 후 채널 확장 예정.

|  | 팀 요청 (광의) | 현재 MVP |
|---|---|---|
| 입력 채널 | 업무 채널 전반 | **카톡만** |
| 운영 환경 | 무관 | **Windows PC 상주** |
| 입력 방법 | 무관 | 노션 Inbox에 텍스트 붙여넣기 |
| 출력 방법 | 무관 | 사전 등록된 카톡방에 자동 메시지 |

## 어떻게 동작하나

### 전체 흐름

```
[1] 매니저가 카톡 대화를 익스포트 (.txt) → 노션 Inbox 페이지에 본문 붙여넣기
        ↓
[2] 시스템이 5분마다 노션 Inbox를 자동 확인
        ↓
[3] AI가 대화에서 「제목 / 내용 / 담당자 / 마감일」 4요소를 분리
        ↓
[4] 노션의 업무 DB에 "검토 대기" 상태로 저장 (중복은 기존 업무에 메모 추가)
        ↓
[5] 매니저가 노션에서 확인·수정·확정 (잘못된 추출 정정 가능)
        ↓
[6] 마감 임박 시 사전 등록된 톡방에 [AI 자동 팔로우업] 표시와 함께 자동 발송
```

전체 흐름이 실제 환경에서 한 번에 동작함을 확인했음 (입력 → AI 분석 → 노션 저장 → 본인 카톡으로 실 메시지 발송).

### 단계별 상세

**[1] 입력 단계**
- 매니저는 카톡 PC 클라이언트의 "대화 내보내기" 메뉴로 톡방 대화를 `.txt` 파일로 저장
- 그 텍스트를 노션의 Inbox DB 새 행 본문에 붙여넣기 (또는 짧은 메모는 노션에 직접 작성)
- "Processed" 체크박스를 해제 상태로 두면 시스템이 처리 대상으로 인식

**[2] 자동 감지 단계**
- 시스템이 백그라운드 상주 (Windows 시작 시 자동 가동)
- 5분 간격으로 노션 Inbox에서 "Processed=false" 인 행을 자동 폴링
- 행의 제목(Memo)뿐 아니라 페이지 본문 블록 전체를 읽어 텍스트로 합침 (긴 카톡 대화 처리용)

**[3] AI 추출 단계**
- 매니저가 선택한 AI 모델에 카톡 대화 텍스트를 전달
- AI에게 "이 4요소로 분리해서 JSON으로 응답하라"고 지시문 부여:
  - **제목** (Title): 한 줄 요약
  - **내용** (What): 구체적인 업무 내용
  - **담당자** (Assignee): 누가 할 일인지 (명시 없으면 "미정")
  - **마감일** (Due Date): 명시된 날짜만 (애매하면 비워둠)
- 추가로 **기존 진행 중인 업무 목록**을 함께 보내서 중복 여부 판단하도록 함

**[4] 노션 저장 단계**
- 신규 업무 → 노션 Tasks DB에 **"검토 대기"** 상태로 새 행 생성
- 기존 업무와 중복으로 판단된 경우 → 새 행 안 만들고, 기존 업무의 Notes 칸에 새 맥락만 timestamp와 함께 추가
- Inbox 행은 Processed=true로 마킹 (다음 사이클부터 처리 대상 제외)

**[5] 매니저 검토 단계** ← 자동화 안 되는 사람의 판단 영역
- 매니저가 노션 Tasks DB에서:
  - Status: "검토 대기" → **"확정"** (또는 "취소")
  - Chatroom: 어느 톡방에 리마인드 보낼지 입력
  - Due Date: 마감일 채우거나 수정
  - AI Followup Enabled: 자동 발송 여부 켜기/끄기 (기본 켜짐)
- 잘못 추출된 항목 수정·삭제 가능. AI는 어디까지나 1차 정리 도우미.

**[6] 자동 발송 단계**
- 시스템이 1분마다 "확정 + 발송 활성화 + 마감 임박" 조건의 업무를 발송 큐에 넣음
- 발송 시점은 마감 24시간 전(D-1) 및 마감 당일(D-day)
- 발송 직전 **화이트리스트 검사** (다음 항목 참조)
- 카톡 PC 자동화로 해당 톡방을 열어 메시지 입력 후 엔터
- 발송 성공/실패는 로컬 DB에 기록, 실패 시 재시도 (최대 3회)

## 사용 도구 · 연결 방식

### 노션
- 업무 대시보드 + Inbox + 화이트리스트 톡방 목록 관리에 사용
- **연결 방식**: Notion API. 매니저가 통합(Integration) 토큰을 발급하고, 사용할 부모 페이지에 통합을 공유한 뒤, 토큰을 환경변수로 등록
- 시스템 첫 실행 시 셋업 스크립트가 Tasks/Whitelist/Inbox 3개 DB를 자동으로 생성
- 데이터 흐름은 모두 노션 ↔ 시스템 사이 직접 통신 (중간 서버 없음)

### 카톡 PC 클라이언트
- 메시지 발송에 사용 (수신은 매니저가 .txt 익스포트 후 노션에 넣는 방식)
- **연결 방식**: 카톡은 공식 API를 제공하지 않음 → PC UI 자동화로 우회
  - Windows 창 핸들 + UI Automation을 결합해 카톡 창을 식별
  - 톡방 검색창에 대상 이름 입력 → 결과 더블클릭 → 새 채팅창 열기
  - 메시지 입력칸에 텍스트 입력 → 엔터로 발송
- 발송 안정성을 위한 6단계 자동화 시퀀스 사용 (탭 진입 → 창 스냅샷 → 톡방 열기 → 제목 검증 → 입력 → 발송)

### AI 모델 
1. **클라우드 AI (Anthropic 모델 - Claude, 외에도 적절한 방식이 있다면 그걸 사용할 예정)**
   - 정확도 가장 높음
   - 연결 방식: Anthropic API 키 또는 Claude Code CLI를 통해 호출
   - 카톡 본문이 Anthropic 서버로 전송됨 → 회사 정보 보안 검토 필요
2. **PC 내부 AI (Ollama 기반 로컬 LLM)**
   - 본인 PC에서만 처리
   - 연결 방식: Ollama 데스크탑 앱을 설치하면 localhost에 자동으로 AI 서버가 뜸. 시스템은 그 주소에 직접 연결
   - 현재 검증 중인 옵션. 처리 시간은 CPU 환경에서 3,000자당 약 1분

### Windows 시작 프로그램
- PC 부팅 시 시스템이 자동으로 백그라운드 가동
- 매니저가 매번 수동 실행할 필요 없음
- 작업 스케줄러보다 간단한 .bat 파일 방식 사용

### 로컬 DB (SQLite)
- 발송 대기 큐, 발송 이력, 처리한 Inbox 행 기록 등에 사용
- 매니저 PC 안에만 존재, 외부 연결 없음

## 안전장치

### 화이트리스트 (사전 등록 톡방만 발송)
- 노션에 **Whitelist DB**가 별도로 있음
- 매니저가 카톡 톡방 제목을 정확히 적어서 등록, 활성/비활성 토글 가능
- 시스템은 발송 직전에 다음을 모두 확인:
  - 노션 Whitelist에 그 톡방 이름이 **활성 상태**로 등록되어 있는지
  - 카톡에서 실제로 열린 채팅창 제목이 등록된 이름과 **완전 일치**하는지 (오타·동명이인 차단)
  - 검색 후 열린 창이 새로 뜬 창이 맞는지 (창 핸들 스냅샷 비교)
- 한 가지라도 어긋나면 즉시 발송 중단

### 메시지 표시
- 모든 자동 메시지 앞에 `[AI 자동 팔로우업]` 접두어 필수 부착
- 받는 사람이 사람 메시지와 혼동 방지

### 시간 기반 차단
- 마감 6시간 이상 지난 건은 발송 안 함 → 늦은 뒷북 리마인드 방지
- 야간·주말 등 시간대별 발송 차단은 추가 작업 예정

### 매니저 컨펌 요건
- "검토 대기" 상태의 업무는 절대 자동 발송 안 됨
- 매니저가 명시적으로 "확정" + "AI Followup Enabled = 켜짐" 으로 바꿔야만 발송 대상
- 즉 AI 추출 결과가 잘못돼도 매니저가 놓치지 않는 한 오발송 없음

## 한계

- AI가 담당자·마감일을 놓치거나 모호하게 추출하는 경우 있음
- **긴 대화 일부 누락 위험** — 현재 앞부분 3,000자(약 20~30개 메시지)만 분석
- 매니저가 "AI가 모든 업무를 잡았나?" 빠르게 검증할 방법이 부족
- 동일 업무가 여러 톡방에서 재언급되는 중복 케이스의 처리 정확도가 AI 판단에만 의존

## 다음 단계 (카톡 MVP 완성도 ↑)

1. 대화를 **날짜 단위로 쪼개기** — 긴 대화 누락 해결 + 처리 속도 개선
2. **카톡 익스포트 파일을 폴더에 놓기만 하면 자동 처리** — 노션에 매번 붙여넣는 부담 해소
3. **AI에게 한국어 카톡 예시·규칙 추가 학습** — 정확도 향상
4. **추출 결과 한눈에 검토** — 노션에 "이 메모에서 N건 잡힘" 링크 표시
